# คู่มือและรายงาน 8D Report

## 8D Report - การรายงานผลโดยใช้เทคนิค 8D

การเกิดสิ่งที่ไม่สอดคล้องขึ้นในระบบ เช่น ปฏิบัติไม่สอดคล้องกับข้อกำหนดขององค์กร ข้อกำหนดมาตรฐาน หรือ ข้อกำหนดของลูกค้า

จำเป็นอย่างยิ่งที่ต้องดำเนินการขจัดสาเหตุที่แท้จริงของปัญหา

เทคนิคหนึ่งที่อุตสาหกรรมยานยนต์ และ กลุ่มอิเล็กทรอนิกส์นิยมใช้เมื่อแก้ไขปัญหาที่เกิดขึ้นที่สาเหตุแท้จริงก็คือ การใช้เทคนิค 8D และอาศัยเครื่องมือ เช่น 5-Why Analysis มาช่วยในการวิเคราะห์หาสาเหตุ

ซึ่งจะนำไปสู่การแก้ไขปัญหาและป้องกันไม่ให้ปัญหานั้นกลับมาเกิดซ้ำได้อีก เพราะจะส่งผลต่อความเชื่อมั่นโดยรวมของลูกค้าที่มีต่อองค์กร

# D0: Preparing (ขั้นตอนการวางแผน)

## D0: ขั้นตอนการวางแผน

เป็นการเตรียมการ หาข้อมูลที่เกี่ยวข้องกับปัญหานั้น เพื่อรวบรวมข้อมูลเบื้องต้น ก่อนเข้าสู่ 8D process

- เป็นปัญหาใหม่ หรือที่เคยเกิดซ้ำๆ (chronic)

- ความเสี่ยงใหม่ๆ เคยเลือกใช้วิธีการใดแก้ไข เหมาะสมหรือไม่

# คู่มือและรายงาน 8D Report

## Definition

### Correction การแก้ปัญหาเบื้องต้น

**สถานะ:** ปัญหาเกิดแล้ว

ทำให้กลับสู่สภาพเดิม (แต่ยังไม่ทราบสาเหตุที่แท้จริง)

**Ex:** การพิมพ์เอกสาร / เอกสารที่เป็นสี เกินความจำเป็น ทำให้เสียค่าใช้จ่าย

แจ้งผู้ใช้งานให้ set output color เป็นแบบ ขาว-ดำ และ secure print แทน normal print

----

### Corrective Action

**สถานะ:** ปัญหาเกิดแล้ว

แก้ไขปรับปรุงที่สาเหตุต้นตอของปัญหา

แก้ไขป้องกันปัญหา โดยหาวิธีการปรับปรุงเพื่อไม่ให้เกิดซ้ำ

**Ex:** สาเหตุเกิดจากพนักงานไม่รู้วิธีการกำหนดเครื่องพิมพ์ หรือไม่ได้ดูโหมดการพิมพ์ให้เป็น ขาว-ดำ ก่อนพิมพ์งาน พิมพ์งานมาตรวจสอบโดยไม่จำเป็น พิมพ์ออกมาเก็บไว้เฉยๆ

**Ex: How to protect,** SE กำหนดมาตรฐานเครื่องพิมพ์ทุกเครื่อง ให้เป็นมาตรฐานสี ขาว-ดำ กำหนด Password โดยเฉพาะสำหรับทุกหน่วยงาน เพื่อตรวจสอบได้ ให้มีการควบคุมจำนวนการพิมพ์เท่าที่จำเป็น และ กำหนดสิทธิ์บางเครื่องที่จำเป็นต้องใช้เอกสารสีในงาน เป็นต้น

----

### Preventive Action ป้องกันก่อนเกิดปัญหา

**สถานะ:** ปัญหายังไม่เกิด

- จัดการเชิงรุก (Proactive Action)
- หาแนวโน้มความเสี่ยงที่จะทำให้เกิดปัญหา (Potential nonconformity)
- วิเคราะห์ Process/System analysis เพื่อสร้าง safeguards เช่น การใช้ FMEA การจัดทำ Risk Assessment และหามาตรการป้องกันไว้ก่อน ที่ปัญหานั้นจะเกิดขึ้น

**Ex.** Set Technician สอนวิธีการใช้เครื่อง Printer manual ให้ผู้ใช้รายใหม่ๆ ป้องกันใช้ผิดประเภท บำรุงรักษาเครื่องพิมพ์ตามวงรอบ ยืดอายุการใช้งาน

## D4 Root Cause Analysis

### Discipline 4: Root Cause Analysis ระบุสาเหตุแท้จริงของปัญหา

• ระบุสาเหตุที่สามารถเป็นไปได้ทั้งหมด ที่อธิบายว่า ทำไมจึงเกิดปัญหานั้น

• ทวนสอบ โดยทดสอบสมมุติฐานแต่ละข้อ กับลักษณะอาการของปัญหา

• เก็บข้อมูลมายืนยัน หรือทำให้พิสูจน์ได้ (ไม่คลุมเครือ)

เพื่อนำมาวางแผนปฏิบัติการแก้ไขที่สาเหตุที่แท้จริงของปัญหา

### เครื่องมือ ระบุสาเหตุที่แท้จริงของปัญหา

• แผนผังแสดงเหตุและผล หรือ แผนภูมิก้างปลา

• (Cause and Effect Diagram, Fishbone Diagram)

• หลักการ 5Why (Why-why Analysis)

## D5 Identify Corrective Action

### Discipline 5: Identify Corrective Action
**ระบุแนวทางป้องกันปัญหา (อย่างถาวร)**

#### วัตถุประสงค์

• ยืนยันแนวทางที่เลือกมาปฏิบัติเพื่อใช้แก้ไขปัญหานั้น เป็นการแก้ไขที่ถาวร

• สามารถป้องกันการเกิดปัญหาซ้ำได้ โดยที่ลูกค้าไม่ได้รับผลกระทบใดๆ

• วิธีการปฏิบัติที่เลือก ต้องช่วยป้องกันการเกิดปัญหานั้นซ้ำ

#### แนวทางการดำเนินการ

• **Establish the solution.**

• **Verification test / reliability demonstration test required.**

• **Test of**
  - Practicality
  - Feasibility, and cost effectiveness,
  - Capability of manufacturing (machine, people, etc.) and supplier to prevent recurrence problem.

• **Deadline required.**

• **Training Plan in case of needs.**

## Review, Verification, Validation

### Review การทบทวน

ทบทวน Input ที่จะนำมาใช้ให้เหมาะสม

เช่น ได้ข้อมูล Spec. DWG จากลูกค้ามา เรียกประชุมทีมเกี่ยวข้องว่าสามารถทำได้หรือไม่

[หาวัตถุดิบได้ สามารถผลิตตาม Spec. ใน DWG ทำได้ตามจำนวน ตามเวลา และคุ้มทุน พื้นที่จัดเก็บเพียงพอ กำลังการผลิตเพียงพอ (จัดเป็น Feasibility Team เพื่อ Review)]

### Verification การทวนสอบ

ทวนสอบผลของ Output (Product) เปรียบเทียบกับ Input (Spec.)

เช่น วัดดูว่า Product ที่ผลิตออกมา ได้ตาม Spec./ ตามข้อกำหนดของลูกค้า หรือไม่

### Validation การทดสอบเพื่อยืนยันผล

ทดสอบยืนยันผลการใช้งาน Output (Product) ได้ตามที่ลูกค้าต้องการ

ทดสอบดูว่า Output (Product) มี Performance ต่างๆ ตามที่ลูกค้าต้องการหรือไม่

เพื่อยืนยันว่าถ้าลูกค้านำไปใช้งานแล้วจะไม่เกิดปัญหา สามารถนำไปประกอบได้ และทำงานได้เป็นปกติ เช่น ทดสอบค่าความแข็ง ค่าความกัดกร่อน ฯลฯ

FPC เช่น ANC ทดสอบ

- **Physical**: Peel strength, tensile, folding endurance, drop test of packaging
- **Chemical**: ค่า NaCl (โซเดียมคลอไรด์) ที่ specimen (ชิ้นงานตัวอย่าง)
- **Electrical**: Resistance, circuit short after apply voltage
